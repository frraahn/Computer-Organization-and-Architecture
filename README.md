# 💻 Computer Organization and Architecture

Welcome to my repository for the **Computer Organization and Architecture (COA)** course at Universiti Teknologi Malaysia (UTM). This space serves as a portfolio documenting our team-based system benchmarking and assembly language project alongside my individual practical database laboratory scripts.

---

## 📂 Repository Structure & Portfolio Breakdown

### 🔹 Project: Hardware Evaluation & Assembly Optimization Benchmarking (Group Team Portfolio)
* **General Summary:** As a team, we analyzed how computer hardware and low-level software work together. First, we tested and compared different computer processors using benchmarking tools like CPU-Z, Geekbench, and Task Manager. Second, we wrote and ran custom Assembly x86 scripts to see how calculation speed and processor workload scale when running repetitive mathematical equations.
* **Core Deliverables:**
  * CPU Hardware Performance Benchmarking Report 
  * Algorithmic Execution Time & ASM Implementation Analysis 
  * Live Interactive System Demo Video 

### 🔹 Lab Work: Low-Level Assembly Programming & Flowchart Engineering
* **General Summary:** A collection of individual hands-on lab exercises focusing on basic x86 assembly language programming and logic planning. The work involved writing programs to handle registry calculations (like multiplication and division), building user-friendly interactive menus using the Irvine32 library, and creating detailed visual flowcharts to plan out our software logic before writing code.
* **Core Deliverables:**
  * Low-Level Arithmetic Operations & Registry Manipulation 
  * Irvine32 Link Library Integration & Branching Menu Systems 
  * Workflow Logic & Algorithmic Blueprinting Maps

## 💡 Course Reflection & Key Takeaways
This course helped me understand exactly how software instructions interact with actual computer hardware. Moving away from high-level languages down to x86 assembly showed me how data moves step-by-step through internal registers like EAX and EBX. Writing scripts for basic math and building interactive choice menus with the Irvine32 library gave me solid practice in managing memory and logic loops at a low level. Additionally, testing different computers during the benchmarking project showed me how hardware layout directly impacts real-world processing speeds. Overall, combining these low-level coding labs with hands-on hardware testing has given me a much better foundation for understanding how computers execute code efficiently.
